üîç The Grep Pipeline Problem

Traditional grep workflow for code analysis:

  1. Search: Use grep to find files containing a pattern
  2. Read: Read the ENTIRE content of all matching files
  3. Process: Send all file contents to the LLM

This results in sending 21,332,560 tokens when only 817 tokens were actually needed!
